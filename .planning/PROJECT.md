# K-Progress — 프로젝트 컨텍스트

## 프로젝트 개요

- **프로젝트명**: K-Progress (케이-프로그레스)
- **한줄 정의**: 금강공업 건설사업부의 재무·원가·수익성 성과 관제탑
- **담당 PL**: 김은지 (바이브코더)
- **시작일**: 2026-05-20

## Core Value (핵심 가치)

> "우리 건설사업부가 지금 얼마를 벌고 있고, 앞으로 얼마를 남길 수 있는가?"

보통 건설 시스템은 현장 사진·도면 위주이나, K-Progress는 **철저히 돈(재무·원가·매출·수익성)에만 집중**한다.  
공무팀·관리팀·회계팀 모두 **같은 화면에서 같은 숫자**를 보고 소통하는 것이 핵심.

## 문제 정의 (Why)

| # | 문제 | 영향 |
|---|------|------|
| 1 | 계약서·실행원가 서류가 담당자 PC에만 있음 | 데이터 자산화 불가, 인수인계 어려움 |
| 2 | 진행률 기준 매출 계산이 복잡해 매번 엑셀 수동 계산 | 시간 낭비 + 계산 실수 발생 |
| 3 | 팀마다 다른 숫자로 소통 | 회의 시 기준 불일치, 의사결정 지연 |

## 최종 목표

1. 계약서·원가 서류를 한 곳에 모으고 금액 변경 이력(변경계약) 관리
2. ERP 데이터 연동 → "계약금액 × 진행률" 자동 계산 → 월 매출 한눈에 확인
3. 준공 프로젝트 데이터 영구 보존 및 지식 자산화

## 기술 스택 (확정)

| 레이어 | 기술 | 결정 근거 |
|--------|------|-----------|
| Frontend | Python + Streamlit | 빠른 구현, 데이터 시각화 강점 |
| K-Progress DB | **Supabase (PostgreSQL)** | 멀티유저 동시 접속 지원, 파일 스토리지 내장 |
| ERP 데이터 | MSSQL ← SAC API | 기존 ERP 직접 수정 금지, API 경유 읽기 전용 |
| 파일 저장 | Supabase Storage | PDF·Excel 첨부파일 보관 |
| 배포 | 사내망 공유 서버 | 팀 전체 URL 단일 접속 |

> Phase 1에서 SQLite로 구현한 기초를 Phase 2에서 Supabase로 마이그레이션한다.

## 아키텍처

```
[공무팀 / 관리팀 / 회계팀]
         ↓  사내망 URL
[K-Progress — Streamlit 앱 (사내 서버)]
         ↓              ↓
   [Supabase]      [SAC API]
  (K-Progress        ↓
   데이터 +      [MSSQL ERP]
   파일 저장)
```

## 사용자 및 접근 방식

- **대상**: 건설사업부 내 공무팀·관리팀·회계팀
- **접근**: 사내망 → 단일 URL → 브라우저 접속 (설치 불필요)
- **데이터 입력**: Excel 일괄 업로드 + 수동 직접 입력 병행
- **파일 관리**: PDF·Excel 파일 업로드 + 숫자 데이터 병행 관리

## 핵심 비즈니스 로직 (변경 금지)

```
진행률 역산 공식 (K-IFRS 15호 기준):
① 예상실행원가 = 직접비 + 간접비
② 진행률 = 예상실행원가 / 실행예산
③ 시스템 매출액 = 계약금액 × 진행률
④ 시스템 수익률 = (계약금액 - 실행예산) / 계약금액  ← 고정값
⑤ 경고 = 시스템 수익률 < 목표 수익률 (default 8%)
```

## Requirements

### Validated (Phase 1 완료)

- ✓ 진행률 역산 엔진 구현 (calculate_project_metrics)
- ✓ 종합 성과 관제탑 대시보드 (탭1)
- ✓ 경고 트리거 (수익률 미달 현장 표시)
- ✓ 매출액 비교 차트 (시스템 역산 vs ERP vs 수금)

### Active (Phase 2~6 대상)

- [ ] Supabase 마이그레이션 (SQLite → PostgreSQL)
- [ ] 프로젝트별 계약 이력 관리 (변경계약 히스토리)
- [ ] 직접비·간접비 항목별 상세 입력·조회
- [ ] Excel 일괄 업로드 (직접비·간접비 양식 파싱)
- [ ] 파일 첨부 (계약서 PDF, 원가 Excel → Supabase Storage)
- [ ] 준공 프로젝트 아카이브 (데이터 영구 보존)
- [ ] Lessons Learned 등록·조회·검색
- [ ] SAC API 연동 (ERP 실데이터 자동 갱신)
- [ ] 사내망 배포 및 팀 접근 설정

### Out of Scope (의도적 제외)

- 현장 사진·도면 관리 — 핵심 차별점 훼손 (돈에만 집중)
- 공정 관리 (바차트·S커브) — 별도 시스템 영역
- 모바일 앱 — 브라우저 반응형으로 대응
- 급여·인사 관리 — ERP 영역

| 결정 | 근거 | 날짜 |

| Python + Streamlit | 빠른 구현, 비개발자 진입 장벽 낮음 | 2026-05-22 |

| Excel 일괄 업로드 우선 | 기존 팀 워크플로우 수용, 적응 장벽 최소화 | 2026-05-29 |
