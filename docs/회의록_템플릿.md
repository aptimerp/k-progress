# 회의록 SVG 템플릿 가이드

회의 내용을 아래 양식으로 정리해 전달하면 SVG 회의록을 자동 생성합니다.

---

## 입력 양식

```
회의일시
YYYY.MM.DD(요일)

참석자
이름1, 이름2, 이름3

회의장소
장소명

개요
회의 목적 및 배경 (2~3줄)

주요 논의 주제: [토픽 제목]
내용 (불릿 형태로)

주요 논의 주제: [토픽 제목]
내용

...

결론 및 운영 원칙
결론 내용
```

---

## SVG 레이아웃 구조

| 영역 | 내용 |
|------|------|
| 헤더 | 금강공업 ERP Track 2 브랜딩 + "회의록" 타이틀 |
| 메타 카드 | 일시 / 참석자 / 장소 / 구분 |
| 개요 박스 | 회의 목적 한 줄 요약 + 세부 논의 항목 |
| 토픽 카드 | 번호 뱃지 + 색상 구분 + 우측 상태 뱃지 |
| 결론 카드 | 진남색 배경 + 주황 포인트 |
| 푸터 | 금강공업 AX팀 · 날짜 |

### 토픽 색상 규칙 (순서대로 순환)
1. 파란색 `#2563EB` — 데이터·취합·기준
2. 보라색 `#7C3AED` — 로직·계산·배부
3. 초록색 `#059669` — 이체·이동·조직
4. 주황색 `#E8922A` — 일정·개발·계획
5. 빨간색 `#DC2626` — 한계·이슈·제약
6. (6개 이상 시 1번부터 재순환)

### 상태 뱃지 종류
- `합의 완료` — 파란 배경
- `진행 중` — 보라 배경
- `검토 중` — 초록 배경
- `요구 확정` — 주황 배경
- `후순위` — 빨간 배경
- `미결` — 회색 배경

### 특수 시각 요소 (토픽 내용에 따라 자동 삽입)
- **흐름 다이어그램**: 로직·배부 흐름이 있을 때 (예: 전체비용 → 직접비 → 공통비)
- **방향 화살표**: 이체·이동 방향이 있을 때 (예: 진천 → 보은)
- **타임라인**: 일정·마일스톤이 있을 때 (예: 현재 → 8월 → 10월)

## 파일 저장 규칙

| 항목 | 규칙 |
|------|------|
| 파일명 | `회의록_YYYYMMDD.svg` |
| 저장 위치 | `docs/` 폴더 |
| 크기 | viewBox `900 × (토픽 수 × 약 170 + 500)` px 자동 조정 |

---

## 사용 예시

> "회의내용 요약한건데 회의록 작성해서 시각화해줘(SVG)"
> → 위 입력 양식 형태로 내용 붙여넣기
> → `docs/회의록_YYYYMMDD.svg` 자동 생성 + GitHub 커밋 요청 시 푸시
