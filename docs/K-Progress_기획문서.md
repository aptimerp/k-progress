# K-Progress 프로젝트 기획문서

> 금강공업 건설사업부 · 프로젝트 실행관리 및 성과 관제탑 시스템  
> 담당 PL: 김은지 · 최초 작성: 2026-06-08

---

## 1. 한 줄 정의

> **"우리 건설사업부가 지금 얼마를 벌고 있고, 앞으로 얼마를 남길 수 있는가?"**

현장 사진·공정표가 아닌 **재무·원가·매출·수익성 숫자**에만 집중하는 성과 관제탑.  
공무팀·관리팀·회계팀이 **같은 화면에서 같은 숫자**를 보고 소통하는 것이 핵심 목표다.

---

## 2. 문제 정의 (Why)

| # | 현재 문제 | 발생 영향 |
|---|-----------|-----------|
| 1 | 계약서·실행원가 서류가 담당자 PC에만 있음 | 데이터 자산화 불가, 인수인계 어려움 |
| 2 | 진행률 기준 매출 계산을 매번 엑셀로 수동 계산 | 시간 낭비 + 계산 실수 |
| 3 | 팀마다 다른 숫자로 소통 | 회의 시 기준 불일치, 의사결정 지연 |

---

## 3. 목표

1. 계약서·원가 서류를 한 곳에 모으고 **변경계약 이력** 관리
2. ERP 데이터 연동 → **"계약금액 × 진행률" 자동 계산** → 월 매출 한눈에 확인
3. 준공 프로젝트 데이터 **영구 보존 및 지식 자산화**

## 5. 사용자 및 접근 방식

- **대상**: 건설사업부 공무팀·관리팀·회계팀
- **접근**: 사내망 → 단일 URL → 브라우저 접속 (설치 불필요)
- **데이터 입력**: Excel 일괄 업로드 + 수동 직접 입력 병행
- **파일 관리**: PDF·Excel 파일 업로드 + 숫자 데이터 병행 관리

## 6. 기술 스택

| 레이어 | 기술 | 결정 근거 |
|--------|------|-----------|
| Frontend | Python + Streamlit | 빠른 구현, 데이터 시각화 강점, 비개발자 진입 장벽 낮음 |
| K-Progress DB | Supabase (PostgreSQL) | 멀티유저 동시 접속, 파일 스토리지 내장 |

| 파일 저장 | Supabase Storage | PDF·Excel 첨부파일 보관 |
| 버전 관리 | GitHub (aptimerp/k-progress) | master 브랜치 단일 운영 |
| 배포 | 사내망 공유 서버 | 팀 전체 URL 단일 접속 |

> Phase 1은 SQLite로 구현 완료 → Phase 2에서 Supabase로 마이그레이션

## 7. 시스템 아키텍처

```
[공무팀 / 관리팀 / 회계팀]
         │  사내망 URL (브라우저)
         ▼
┌─────────────────────────┐
│  K-Progress             │
│  Streamlit 앱           │
│  (사내망 공유 서버)      │
└────────┬────────────────┘
         │                │
         ▼                ▼
  ┌─────────────┐   ┌───────────┐
  │  Supabase   │   │  SAC API  │
  │ PostgreSQL  │   │  (HTTP)   │
  │ + Storage   │   └─────┬─────┘
  │             │         │
  │ K-Progress  │         ▼
  │ 자체 데이터  │   ┌───────────┐
  │ + 첨부파일  │   │ MSSQL ERP │
  └─────────────┘   │ (기존 ERP)│
                    └───────────┘
```

**데이터 원칙**
- K-Progress 자체 데이터(계약·원가·파일) → Supabase 저장
- ERP 데이터(매출·수금 실적) → SAC API 경유 읽기 전용
- 기존 ERP DB 직접 수정 절대 금지

## 8. 핵심 비즈니스 로직

K-IFRS 15호 기준 **진행률 역산 공식** (변경 불가 고정값)

```
① 예상실행원가  = 직접비 + 간접비
② 진행률       = 예상실행원가 ÷ 실행예산
③ 시스템 매출액 = 계약금액 × 진행률
④ 시스템 수익률 = (계약금액 - 실행예산) ÷ 계약금액   ← 고정값
⑤ 경고 트리거  = 시스템 수익률 < 목표 수익률 (기본 8%)
```
