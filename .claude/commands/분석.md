당신은 구현 진입 전에 기술적 가정·리스크·의존성을 표로 정리합니다. **구현은 하지 않습니다.**

## 언제 쓰나 (필수 아님 — 아래 상황에서 권장)
1. UI 등 초기 작업이 끝나고 기술 스택을 축적하기 시작할 때 (중반부터 쓰기 좋음)
2. 기술적으로 중요하고 큰 페이즈를 진행할 때 (아키텍처 결정 · 외부 연동 등)
3. 기존에 만들어 놓은 것 중에서 대대적인 기술 검증이 필요할 때
4. AI가 쓸데없는 것 미리 만들어 놓지 않도록 방지선이 필요할 때

## 진행 단계
1. **`/gsd:list-phase-assumptions` 명령 호출** — GSD의 가정 나열 워크플로우 진입
2. `.planning/REQUIREMENTS.md` · `PLAN.md` 읽고 다음 표 생성:
   - 🔍 **가정 (Assumption)**: "이게 사실이라고 가정한다" 목록
   - ⚠️ **리스크 (Risk)**: 구현 중 깨질 수 있는 지점
   - 🔗 **의존성 (Dependency)**: 외부 API·라이브러리·타팀 자원
   - 🚫 **쓸데없는 구현 후보**: "만들어도 안 쓰일 가능성 높은 것" 목록 (AI 과잉 개발 방지선)
3. 결과를 `.planning/ANALYSIS.md`로 저장 + 사용자에게 표 표시
4. 사용자가 검토 후 필요시 PLAN.md 수정 권장
5. 완료 안내: "/실행 명령으로 다음 단계 진행"

## 절대 금지
- 코드 작성 금지
- 파일 생성·수정 금지 (ANALYSIS.md 만 예외)
- 외부 API 호출 금지

## 폴백 (GSD 미설치 시)
- REQUIREMENTS.md + PLAN.md 읽고 수동으로 위 4가지 표 작성
- `.planning/ANALYSIS.md` 저장
